# Security Configuration Guide

## ⚠️ IMPORTANT: Never Commit Credentials

This project uses environment variables for all sensitive data. **NEVER commit `.env` files to git.**

---

## Required Environment Variables

### API Server (`api/.env`)

```bash
# OpenClaw Gateway (REQUIRED)
OPENCLAW_GATEWAY_URL=ws://127.0.0.1:18789
OPENCLAW_GATEWAY_TOKEN=<your-token-here>

# Dashboard Authentication
GROOT_DASHBOARD_TOKEN=<strong-random-token>

# Optional
API_PORT=3333
CLAUDE_API_KEY=sk-...
REDIS_URL=redis://localhost:6379
```

### Dashboard (`dashboard/.env.local`)

```bash
NEXT_PUBLIC_API_URL=http://localhost:3333
NEXT_PUBLIC_API_TOKEN=<matches-GROOT_DASHBOARD_TOKEN>
```

## Setup Instructions

### 1. Generate Tokens

**Dashboard Token (use strong random string):**
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

**OpenClaw Token:**
Get from `~/.openclaw/openclaw.json` (generated by OpenClaw setup)

### 2. Create `.env` Files

**API:**
```bash
cp api/.env.example api/.env
# Edit and fill in actual values
nano api/.env
```

**Dashboard:**
```bash
cp dashboard/.env.example dashboard/.env.local
# Edit and fill in actual values
nano dashboard/.env.local
```

### 3. Verify No Secrets in Git

```bash
# Check for accidental commits
git log -p --all | grep -i "token\|password\|secret\|key=" || echo "✅ Clean"

# Check staged files
git diff --cached | grep -i "token\|password" || echo "✅ Clean"
```

---

## Security Checklist

- [ ] All `.env` files are in `.gitignore`
- [ ] Never run `git add .env*`
- [ ] Use `git status` before committing
- [ ] Rotate tokens if accidentally exposed
- [ ] Use `OPENCLAWTOKEN` env var (never hardcoded)
- [ ] Dashboard token is strong (32+ random characters)
- [ ] Don't share `.env` files via email/chat

---

## If Token is Accidentally Exposed

1. **Immediately rotate the token** in OpenClaw config
2. **Update environment variables** on all systems
3. **Use git-filter-repo to remove from history:**
   ```bash
   git filter-repo --invert-paths --path-glob "*.env"
   git push -f origin main
   ```
4. **Force-push to ensure history is clean**

---

## Remote Deployment

When deploying to production:

1. **Use secrets management** (AWS Secrets Manager, Vault, etc.)
2. **Never embed tokens in Docker images**
3. **Use environment variables at runtime**
4. **Rotate tokens regularly**
5. **Audit access logs**

---

## Tools for Secret Detection

Monitor for accidentally exposed secrets:

```bash
# Install TruffleHog (detects exposed tokens)
pip install truffleHog

# Scan repository
truffleHog filesystem . --json

# Scan git history
git log -p --all | grep -i "token\|password\|secret"
```
